What role does Storybook play exactly?

You have git tokens → Storybook stories → Figma. Where does Storybook live in this pipeline?

A

Source of truth for component structure

Stories define which components exist and their variants/props. The n8n workflow reads *.stories.ts files from git and uses that to build Figma frames. Storybook itself is the spec.

B

Visual preview — Figma mirrors what Storybook renders

Storybook renders real React components. An n8n step screenshots each story and embeds the image into Figma as a reference frame alongside the token variables.

C

Both — spec + visual reference

Stories define structure AND n8n screenshots the rendered output. Figma gets native frames (built from token data) + a reference image showing how it actually renders in code.

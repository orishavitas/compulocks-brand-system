How do components get defined?

The tokens tell us colors/fonts/spacing — but buttons need states, variants, sizes. Where does that spec live?

A

Hardcoded in the plugin

Plugin has a built-in set of components (button, card, input, badge, tag) with fixed states and variants. Simple, no extra files needed.

B

New components.json in git

Add a tokens/components.json that defines each component — its variants, states, which tokens it uses. Plugin reads this alongside color/type/spacing.

C

Storybook stories as the spec

Write Storybook stories for each component. The plugin (or an n8n step) reads the stories to understand variants/states and builds matching Figma frames.
